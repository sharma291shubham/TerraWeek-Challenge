# 🌍 TerraWeek — Day 1: Building My Terraform Foundation

This repository contains my work for **Day 1** of the TerraWeek public learning challenge.

The goal of Day 1 was to understand the fundamentals of **Infrastructure as Code (IaC)** before diving deeper into Terraform. Instead of focusing only on commands, I explored the reasoning behind Terraform—why it exists, the problems it solves, and the core concepts that make it one of the most widely used Infrastructure as Code tools.

## 📚 What I Covered

- What is Infrastructure as Code (IaC)?
- Why Infrastructure as Code matters
- What is Terraform?
- Why Terraform is so popular
- Terraform vs. OpenTofu, Pulumi, CloudFormation, and Ansible
- Core Terraform concepts:
  - Providers
  - Resources
  - State
  - Plans
  - HCL
  - Modules
- The Terraform workflow
- Installing Terraform
- Running my first Terraform configuration
- Understanding `.terraform.lock.hcl`

## 💻 Hands-on

During this challenge, I worked with:

- `terraform init`
- `terraform fmt`
- `terraform validate`
- `terraform plan`
- `terraform apply`
- `terraform destroy`

I also explored what each command does behind the scenes and how Terraform uses state and providers throughout the workflow. :contentReference[oaicite:0]{index=0}
