# TerraWeek – Day 2: Writing Reusable Terraform with HCL

Welcome to **Day 2** of my **#TerraWeek** learning journey!

The focus today was understanding **HashiCorp Configuration Language (HCL)** and learning how to write reusable Terraform configurations using variables, locals, outputs, and built-in functions.

## 📚 Topics Covered

* Understanding HCL syntax

  * Blocks
  * Arguments
  * Expressions
* Input variables
  * Types
  * Default values
  * Validation
  * Sensitive variables
* Local values
* Output values
* Built-in functions
* Variable precedence
* `terraform.tfvars`
* `terraform console`

## 💻 What I Built

A simple Docker-based Terraform configuration demonstrating:

* Reusable variables
* Computed local values
* Outputs
* Built-in functions
* Variable validation
* Different Terraform variable types
* Variable precedence using `terraform.tfvars`

## 🧪 Bonus Explorations

Beyond the challenge, I also explored:

* Why Terraform uses HCL instead of YAML
* Using `terraform console` as a playground for HCL expressions
* `for` expressions
* Conditional expressions
* Optional object attributes
* Terraform variable precedence in practice
